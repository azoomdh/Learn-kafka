# ví dụ consumer

để kết nối với cụm kafka = sử dụng API KafkaConsumer

file cấu hình

```properties
bootstrap.servers = broker1, broker2, broker3
```

file ví dụ

```java
Properties config = loadProperties("kafka.properties");


KafkaConsummer<String, String> consummer = new KafkaConsumer<>(config);

consummer.subcribe(List.of("TemperatureData_Topic"));

// chạy mãi mãi
while(true){
    ConsummerRecords<String, String> records = consumer.poll(Duration.ofMillis(10));
    for(ConsummerRecord<String, String> record : records){
        System.out.println("Message=" + record.key() + ":" + record.value() );
        // các bước xử lý khác
    }
}
```

message gồm : 

* key 

* value : nội dung dữ liệu

* phân vùng chứa message

* timestamp : thời gian message được kafka ghi lại

* tiêu đề : siêu dữ liệu tùy chọn

# offset

kafka theo dõi offset của 1 consumer tiêu thụ

đang ở offset=600, nếu consumer chết giữa chừng rồi sống lại, kafka sẽ thông báo offset 600 cho consumer đó 

> ❓(nghi ngờ)
> 
> offset của partition là gắn với group consumer
> 
> * kafka cho `group1.consumer1` quản lý `topic1.partition1`
> 
> * `group1.consumer1` chạy đến `topic1.partition1.offset.group1`=5
> 
> * `group1.consumer1` chết 
>   
>   * kafka cho `group1.consumer2` quản lý `topic1.partition1`
>   * `group1.consumer2` chạy tiếp từ `topic1.partition1.offset.group1`=5
> 
> * `group1.consummer2` chạy đến `topic1.partition1.offset.group1`=12
> 
> * `group1.consumer1` hồi sinh 
>   
>   * kafka cho `group1.consumer1` quản lý lại `topic1.partition1`, 
>   
>   * `group1.consumer1` chạy tiếp từ `topic1.partition1.offset.group1`=12

# nhóm consumer

nhóm consumer A = là cách để đọc song song

quan hệ giữa consumer-partition là 1-N

* 1 consumer có thể đọc nhiều partition

* 1 partition chỉ có thể có 1 consummer đọc

<br/>

nếu có `1 topic-3 partition` và `1 consumer group-4 consumer`

-> thì sẽ có 1 consumer nghỉ, 3 cái còn lại mỗi cái đọc 1 phân vùng

<br/>

nếu 1 consumer gặp lỗi -> kafka sẽ gán phân vùng của consumer lỗi cho consumer khác

`1 consumer chiếm 1 partition`->`1 consumer chiếm 2 partition`

<br/>

2 consumer khác "Consumer group" có thể đọc tin nhắn trên 1 phân vùng song song với nhau mà không bị lỗi gì
